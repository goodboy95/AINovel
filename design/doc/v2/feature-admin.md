# 后台管理系统设计

## 1. 架构
- **独立布局**：使用 `AdminLayout`，采用深色系设计，与前台创作区区分。
- **权限**：严格校验 Admin 角色。

## 2. 核心模块：模型管理
- **目的**：控制 AI 成本与定价。
- **字段**：
  - `InputMultiplier` / `OutputMultiplier`: 核心定价参数。
  - `PoolId`: 关联的 API 池（后端负责池内负载均衡）。
- **前端展示**：表格列出所有模型，支持开关启用/禁用。

## 3. 仪表盘
- **数据源**：`GET /api/admin/dashboard-stats`。
- **关注点**：
  - **积分消耗**：监控系统成本。
  - **API 错误率**：监控上游模型服务稳定性。
  - **待审素材**：内容安全合规。
