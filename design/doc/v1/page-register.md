# 注册页（Register）

- **路由/文件**：`/register`
- **对应设计稿文件**：`src/pages/auth/Register.tsx`
- **布局**：居中卡片；字段：用户名、邮箱、密码、确认密码；底部跳转登录链接。
- **交互流程**：
  - 表单校验：邮箱格式、必填，确认密码与密码一致。
  - 提交 `POST /api/v1/auth/register`，Body: `{ username, email, password }`。
  - 成功：提示“注册成功”并跳转 `/login`；失败显示后端 `message` 或通用错误。
- **依赖/权限**：公开接口，无需 Token。
- **待完善**：
  - 密码复杂度提示；
  - 用户名/邮箱唯一性即时校验；
  - 注册成功后直接登录或邮件验证流程（当前无）。

## 开发对接指南 (Mock vs Real)

### 1. 注册逻辑
- **当前 Mock**：`api.auth.register` 仅模拟延迟后返回 `{ success: true }`，不进行任何数据存储或校验。
- **真实对接**：
  - 对接 `POST /api/v1/auth/register`。
  - 后端需处理用户名/邮箱查重。前端应捕获 `409 Conflict` 错误并提示用户“该用户名/邮箱已被注册”。

### 2. 表单验证
- **当前 Mock**：仅前端简单的非空和密码一致性校验。
- **真实对接**：
  - 建议引入 `zod` 或后端验证规则，对密码强度（大小写、数字、符号）进行实时校验。
